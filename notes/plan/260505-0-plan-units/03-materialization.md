# Unit 03 — Plan dir materialization

**Blocked by:** 02
**Agents involved:** main only
**Topology:** none

## Goal

Turn a validated `Plan` into a directory of files on disk. This is the new core behavior. Markdown only in this unit — `overview.html` lands in unit 04.

## Tasks

### Path resolution

New module `src/materialize.rs` (or extend `output.rs`):

- `resolve_target_dir(plan: &Plan, plans_root: &Path, today: NaiveDate) -> Result<PathBuf>`
  - Format: `<plans_root>/<YYMMDD>-<N>-<slug>`
  - YYMMDD: `today.format("%y%m%d")`
  - N: scan `plans_root` and sibling dirs (`research/`, `backlog/`, `done/plan/` if present, per `notes/AGENTS.md` shared-counter rule) for entries matching `^<today>-(\d+)-`. Take max + 1, or 0 if none.
  - `plans_root` for v1: `<project_root>/notes/plan/`. Project root = `$CLAUDE_PROJECT_DIR` (research finding: hooks fire with `cwd=~`); fall back to PWD with stderr warning if unset.

### File generation

- `write_overview_md(plan: &Plan, dir: &Path)` — front-matter + decisions skeleton + unit table (use this very plan's `overview.md` as the template).
- `write_progress_md(plan: &Plan, dir: &Path)` — cursor pointing at `units[0].id`, empty Done, empty Blockers, and the resume-protocol note line.
- `write_unit_md(plan: &Plan, unit: &Unit, dir: &Path)` for each unit:
  - Filename: `<unit.id>.md` (e.g. `01-housekeeping.md`). Note: `unit.id` already includes the `0N-` prefix.
  - Body: title H1, metadata block (blocked-by + agents + topology presence), then `unit.body_markdown` verbatim, then a `## Review` checklist generated from `review_steps`, then a footer line pointing to progress.md.
  - If `unit.topology` is `Some`: emit a ` ```mermaid ` block beneath the body, generated by reusing `mermaid::render_topology` in unit-scope mode (no surrounding "Phase 1/2" framing — it's just one diagram for the unit).

### Idempotency

- If target dir already exists: error with clear message, do not clobber. `--force` flag deferred.
- Within target dir, write atomically: write to `<file>.tmp`, rename. Don't leave half-written files if the process is killed.

### CLI surface

- New subcommand: `planview materialize <plan.json>` — manual invocation, mainly for testing. Behavior matches the hook path.

### Tests

New `tests/materialize_test.rs`:

- Fixture plan JSON → materialized dir → assert each file's contents match a checked-in golden file. Use a tempdir for the run, copy the result for diff.
- Plan with one unit carrying topology → unit md contains the expected mermaid block (substring match, not full snapshot).
- Path resolution: synthetic `plans_root` containing `260505-0-other` → next dir is `260505-1-<slug>`.
- Path resolution: empty `plans_root` → next dir is `260505-0-<slug>`.
- Existing target dir → error.

## Acceptance

- Golden-file tests pass.
- Manual smoke: hand-craft a Plan JSON, run `planview materialize <file>`, inspect dir.
