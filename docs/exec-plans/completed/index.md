# exec-plans/completed/

Finished plans — **frozen historical records**, not current truth about the code.

- **Status: archived / immutable.** Don't garden these; freezing is the point. A completed plan documents *what was intended and done*, dated to its era — the code may have moved on since.
- **Provenance.** Each plan is stamped on archive — `STATUS: completed · <date> · realized-by <commit>` — so a reader knows when it was true and where to find the realization.
- **Reading rule.** Treat code references inside as *historical*. The convention is "reference, don't paste" (`path:symbol`, not frozen snippets), precisely so these records age gracefully instead of turning into confidently-wrong stale code.
- **Naming:** `YYMMDD-N-slug/` — the id the plan carried in `../active/`.
