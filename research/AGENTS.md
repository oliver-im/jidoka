# research/

Pre-implementation research documents for planview. These are point-in-time analyses — verify claims against current state before acting on them.

## Documents

- **[strategic-review.md](strategic-review.md)** — Evaluation of four strategic risks (competitive overlap, native Claude visualization, plan mode longevity, agent-agnostic design). Conclusion: proceed with implementation.
- **[tech-stack.md](tech-stack.md)** — Language/runtime comparison (Rust, Go, Bun, Deno, Swift, Zig) for the renderer binary. Evaluates binary size, startup time, JSON handling, MCP SDK availability, cross-compilation. Conclusion: Rust wins on pure technical merits, Go close second, Bun is pragmatic but technically wasteful.
- **[cli-vs-mcp.md](cli-vs-mcp.md)** — Whether planview needs an MCP server for agent-agnosticism. Analyzes Poehnelt and Holmes arguments for CLI-first design. Conclusion: the CLI binary is already the agent-agnostic interface; defer MCP to post-v1. Downgrades strategic review recommendation #2 and reduces MCP SDK weight in tech stack analysis.
- **[googleworkspace-cli.md](googleworkspace-cli.md)** — Reference project review of `googleworkspace/cli` (Rust CLI for humans + agents). Transferable lessons: CLI-first product framing, self-describing flags, skills as thin adapters, early distribution mindset. Confirms cli-vs-mcp.md conclusions.
- **[diagram-rendering.md](diagram-rendering.md)** — Whether Mermaid is the right diagram rendering backend. Evaluates 8 candidates (Mermaid, Graphviz WASM, D2, ELK.js, Cytoscape.js, vis-network, Kroki, Excalidraw). Conclusion: Mermaid is correct for V1 — only option with native stadium/pill + double circle shapes matching planview's visual language. Revises developer guide's D2 assessment (8.2 MB bundle, missing shapes, TALA unavailable in browser). Graphviz WASM (778 KB, gold-standard layout) is the strongest upgrade path if Mermaid limitations become blocking.
- **[skill-distribution.md](skill-distribution.md)** — How to maintain one canonical skill source while distributing to Claude Code, Codex, and Cursor. Reviews official skill discovery paths, the practical downsides of symlink-based installs, copy/sync-based distribution patterns, and a three-layer local testing strategy (static validation, sync validation, runtime smoke tests). Conclusion: keep one canonical source, generate/copy agent-specific installs, and treat distribution/testing as first-class product work.
